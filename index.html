Project Proposal

  1. Product Name: playlistDB
  2. Team Members: Marcel
  3. The purpose of this Database is to separate a list of unorganized songs from my soundcloud likes into three separate tables containing information about the songs, albums, and artists. When creating the database the initial attributes will be basic such as Album name, Artist Name, song Length, and song name. When writing queries and adding information to the Database I plan to add attributes such as links to songs, artist profiles, artist label affiliation,music videos, links to lyrics, and album release dates. I will push the code to github and deploy it on microsoft azure with as a web application.
  5. Proposal Last Updated (12/5/21) This is the final draft
  6. Feature List

      7. Songs

    • There will be a table which holds the entire list of all the songs titled SoundCloud Likes. The fields will be approximately 100 at first. The initial Attributes of the songs will simply be artist, genre, length of song, album, and song ID. A unique ID will be automatically generated for every song entered into the database. This will be the primary key for this table as well and the foreign key will be Album which references the album ID. Album cannot be null as well as the name of the song. Every Album will match an ID in the album table. Later on links to songs, music videos, and lyrics will be added with a query.

      • Albums

    • The Albums table will initially contain information such as album name, length, artist, and album ID. The Primary Key for this table will be album ID and the foreign key will be artist which references the artist ID. The album and artist name may not be null, the artist must match an artist ID. Additional attributes will be added with a query after creating the database such as release data

      • Artists

    • Initially the Artist table will hold information such as artist Name, age, and Artist ID. The name of the artist cannot be null and each artist will be assigned a unique ID. The primary key for this table will be the artist ID. Later attributes such as links to the artist profiles and label affiliation will be added with queries.
